## Goal

Land a zero-IO `cbscore-types` crate with the full descriptor / config / error
surface specified in design 002. Round-trip tests against a curated corpus of
real-world YAML/JSON files form the M0 acceptance gate. Empty `cbscore` and
`cbsbuild` crates are added in the same phase so that subsequent phases can fill
them in without revisiting the workspace `Cargo.toml`.

End state: `cargo build --workspace` succeeds; `cargo test -p cbscore-types`
passes, covering both round-trip directions — Rust-value-side
(`create → store → load == create`, per `cbsd-rs/docs/cbscore-rs/CLAUDE.md` §
Correctness Invariants item 1) on the hand-crafted values introduced in Commits
3 and 4, and file-side (`load → store → load == load`) on the real-world
YAML/JSON corpus introduced in Commit 5 across every wire format named in design
002 § Wire-Format Versioning; the binary `cbsbuild` prints a placeholder string
and exits 0.

## Depends on

## Out of scope

- Any IO. `cbscore::config::Config::load` and secrets-manager IO land in Phase
  3; `cbscore::versions::desc::{read,write}_descriptor` lands in Phase 4 (the
  runner is the first consumer, but the read side is also reused by Phase 6's
  `cbsbuild versions show`); descriptor-store walks land in seq-004 (Phase 6).
  cbscore-types only exposes the `serde`-derived types and pure functions.
- Design-001-specific concerns beyond the workspace member adds (the lift-out
  invariants, Cargo dep allowlists, etc.). Those remain authoritative in design
  001 and can be revisited in a separate seq-001 plan if needed.
- The `Config.paths.versions: Option<Utf8PathBuf>` field. Owned by seq-004
  (design 004 OQ1 Step 1); lands in Phase 6 alongside the `--versions-dir` CLI
  flag and the resolver. Phase 1's `PathsConfig` ships without it.
- `S3Error` (wrapping `aws_sdk_s3` framework errors) and `VaultError` (wrapping
  `vaultrs::error::ClientError`) are intentionally absent from Phase 1's error
  taxonomy. Both are framework-error wrappers per design 002 §Error Taxonomy
  lines 239–240 and land in Phase 3 alongside the IO modules that produce them
  (`utils::s3` and `utils::vault`, respectively). They live in `cbscore`, not
  `cbscore-types`, matching the `GitError` placement pattern from Phase 2.
- `cbscore::versions::utils` parsing (`parse_version`, `get_version_type`,
  `get_major_version`, `get_minor_version`, `normalize_version`,
  `parse_component_refs`). All six functions are pure but live in the `cbscore`
  library crate, not `cbscore-types`, because each (directly or transitively via
  `parse_version`) depends on the `regex` crate — allowed in `cbscore` per
  design 001 § Cargo Sketch, intentionally absent from `cbscore-types`.
  cbscore-types ships only the `VersionType` enum and the `MalformedVersion`
  error variant; the full parse family lands in Phase 2. **This is a drift from
  design 002 § Version Descriptors & Creation (which places all six signatures
  under `cbscore-types::versions::utils`); design 002 should be updated to match
  — flagged for a follow-up edit, not blocking on this plan.**

## Commit 2 — error taxonomy + logger module

Add the `thiserror`-derived error enums per design 002 § Error Taxonomy plus the
tracing target-hierarchy module per design 001. All errors and the logger live
in `cbscore-types` so callers across workspace members (including external
Python-replacement crates like `cbc`) can match on them and configure logging
without depending on the IO library.

**Files:**

- `cbsd-rs/cbscore-types/src/errors.rs` — root `CbsError` enum (or marker trait)
  covering the cross-cutting variants (`MalformedVersion`, `NoSuchVersion`,
  `UnknownRepository`).
- `cbsd-rs/cbscore-types/src/logger.rs` — tracing target hierarchy constants
  (`pub const TARGET_*: &str = "cbscore"`, `"cbscore::runner"`,
  `"cbscore::builder"`, …) plus `pub fn set_debug_logging()` that configures a
  `tracing_subscriber::EnvFilter`. Per design 001 §Workspace Layout line 100 and
  §Crate Responsibilities lines 218–219. Zero IO; the module is published from
  `cbscore-types` so external consumers (`cbc` imports
  `logger.set_debug_logging` per design 001 §Downstream Consumers) can pull it
  in without depending on `cbscore`.
- `cbsd-rs/cbscore-types/src/versions/errors.rs` — `VersionError` with
  `InvalidDescriptor`, `NoSuchDescriptor`, `MissingSchemaVersion`,
  `UnknownSchemaVersion { found, max_supported }`, and the future home of
  `MalformedVersion` references. The two schema_version variants parallel
  `ConfigError::MissingSchemaVersion` / `UnknownSchemaVersion` and surface from
  `cbscore::versions::desc::read_descriptor` (Phase 4 Commit 1) when the JSON
  file lacks or over-shoots the `schema_version` key.
- `cbsd-rs/cbscore-types/src/config/errors.rs` — `ConfigError` including
  `MissingSchemaVersion` and `UnknownSchemaVersion { found, max_supported }` per
  design 002 § Wire-Format Versioning.
- `cbsd-rs/cbscore-types/src/utils/secrets/errors.rs` — `SecretsError` with
  `Manager` variant.
- `cbsd-rs/cbscore-types/src/utils/podman/errors.rs` —
  `PodmanError { retcode, stderr }`.
- `cbsd-rs/cbscore-types/src/utils/buildah/errors.rs` — `BuildahError`.
- `cbsd-rs/cbscore-types/src/utils/subprocess/errors.rs` — `CommandError` (with
  `Timeout { after }` per design 002 § Subprocess & Secret Redaction).
- `cbsd-rs/cbscore-types/src/releases/errors.rs` — `ReleaseError`.
- `cbsd-rs/cbscore-types/src/images/errors.rs` — `ImageDescriptorError`.
- `cbsd-rs/cbscore-types/src/containers/errors.rs` — `ContainerError`.
- `cbsd-rs/cbscore-types/src/builder/errors.rs` — `BuilderError` (with
  `MissingScript` variant).
- `cbsd-rs/cbscore-types/src/runner/errors.rs` — `RunnerError` with variants
  `Cancelled` (SIGTERM / outer cancellation), `Timeout` (the outer runner-level
  `tokio::time::timeout` elapsed — the whole-runner budget, 4 hours by default
  per design 002 line 849; **not** the per-subprocess `CommandError::Timeout`
  which surfaces via `#[from]` as `Command(CommandError)` below), and
  `Command(CommandError)` / `Podman(PodmanError)` wrappers via `#[from]` for
  subprocess and podman-stop failures. Phase 4 Commit 3's runner pipeline is the
  primary consumer; see its §Design constraints "Two-layer timeout architecture"
  bullet for the wrapping rule.
- `cbsd-rs/cbscore-types/src/lib.rs` — module declarations + public re-exports.

**Design rules** (from design 002 § Error Taxonomy):

- One `thiserror` enum per subsystem; `#[from]` for wrap-and-match.
- `Display` impls match the Python `__str__` output so CLI stderr stays visually
  identical.
- Variants never carry boxed `dyn Error` unless the underlying error is a
  framework error (`reqwest`, `aws_sdk_s3`, …) that cannot be exhaustively
  matched. Those wraps land in Phase 3 alongside the IO modules that produce
  them.

**Testable:** `cargo build --workspace`. Each error enum compiles with
`#[derive(Debug, Error)]`. Doctests on a representative `Display` impl assert
the exact Python-equivalent message string.

## Commit 3 — config + secrets types

Port the config and secrets type surface per design 002 § Configuration &
Secrets Subsystem.

**Files:**

The `config/` subtree mirrors design 001 §Workspace Layout lines 101–105 — four
files, not a single `mod.rs`. The split groups related substructs and keeps each
file small enough to scan.

- `cbsd-rs/cbscore-types/src/config/mod.rs` — top-level `Config`,
  `SigningConfig`, `LoggingConfig`. Re-exports the sub-module types so callers
  can `use cbscore_types::config::{Config, PathsConfig, …}` uniformly.
- `cbsd-rs/cbscore-types/src/config/paths.rs` — `PathsConfig` (`components`,
  `scratch`, `scratch_containers`, `ccache`).
- `cbsd-rs/cbscore-types/src/config/storage.rs` — `StorageConfig` and any nested
  storage backends per design 001 ("`S3StorageConfig`, …").
- `cbsd-rs/cbscore-types/src/config/vault.rs` — `VaultConfig`,
  `VaultUserPassConfig`, `VaultAppRoleConfig`.
- All structs serde-derived with `#[serde(rename_all = "kebab-case")]` at the
  container level. `camino::Utf8PathBuf` for path fields.
- `cbsd-rs/cbscore-types/src/utils/secrets/mod.rs` — top-level `SecretEntry` +
  the three credential families:
  - `GitCreds` (tagged on `creds`: `plain` / `vault`) with `GitPlainCreds` and
    `GitVaultCreds` (each tagged on `type`: `ssh` / `token` / `https` per design
    002 § Git secrets) — **explicit `type:` discriminator is a deliberate format
    choice; deployed Python `secrets.yaml` files lack it and operators re-tag at
    cutover.**
  - `SigningCreds` (tagged on `type` — deployed files already carry the field).
  - `RegistryCreds` (tagged on `creds` — single leaf per outer value).

**Design constraints:**

- `Utf8PathBuf` requires `camino` with the `serde1` feature in
  `cbscore-types/Cargo.toml`. The dep belongs on the types crate side because
  the `Utf8PathBuf` fields participate in `#[derive(Serialize, Deserialize)]`
  directly (design 002 line 472–475).
- `#[serde(rename_all = "kebab-case")]` applies at the **container** level for
  config / secrets / vault YAMLs, NOT for descriptors. Review every new struct
  against the wire-format split before adding the attribute (design 002 §
  Wire-Format distinction across the codebase, lines 665–670).

**Testable:** Per-type round-trip property tests
(`create → store → load == create`, i.e. Rust value → serialize → deserialize,
value-equal per `cbsd-rs/docs/cbscore-rs/CLAUDE.md` § Correctness Invariants
item 1) on hand-crafted Rust values. Negative tests: untagged `git` entry
rejected with a serde error naming the missing `type:` field.

## Commit 4 — descriptor types

Port the descriptor type surface — version, container, image, release,
build-report, core-component.

**Files:**

- `cbsd-rs/cbscore-types/src/versions/desc.rs` — `VersionDescriptor`,
  `VersionSignedOffBy`, `VersionImage`, `VersionComponent`. **NO `rename_all`
  attribute** — keys stay snake_case (design 002 line 644–647).
- `cbsd-rs/cbscore-types/src/versions/utils.rs` — `VersionType` enum only
  (`#[serde(rename_all = "lowercase")]`: `Release`, `Dev`, `Test`, `Ci`). The
  parse-family signatures (`parse_version`, `get_major_version`,
  `get_minor_version`, `normalize_version`, `parse_component_refs`) live in
  `cbscore::versions::utils` and land in Phase 2 — see § Out of scope above.
- `cbsd-rs/cbscore-types/src/containers/desc.rs` — `ContainerDescriptor`
  (snake_case keys).
- `cbsd-rs/cbscore-types/src/images/desc.rs` — `ImageDescriptor`.
- `cbsd-rs/cbscore-types/src/releases/desc.rs` — `ArchType`
  (`#[serde(rename_all = "snake_case")]`), `BuildType`
  (`#[serde(rename_all = "lowercase")]`), `BuildInfo`, `ReleaseComponentVersion`
  (flattened `ReleaseComponentHeader + BuildInfo` per Python's pydantic
  multi-inheritance), `ReleaseDesc`, `ReleaseArtifacts`.
- `cbsd-rs/cbscore-types/src/builder/report.rs` — `BuildArtifactReport` (renamed
  from the Python `BuildArtifactReport.report_version`; the field is now
  `schema_version` per design 002 § Wire-Format Versioning).
- `cbsd-rs/cbscore-types/src/core/component.rs` — `CoreComponent`,
  `CoreComponentLoc`.

**Design constraints:**

- Descriptor structs are **JSON with snake_case keys** — no `rename_all`. The
  split is a hard invariant; review every new struct against design 002 lines
  665–670 before deciding.
- The schema-version marker key follows the host struct's casing rule, per
  design 002 §Wire-Format Versioning §Interaction with other wire-format
  decisions: snake-case descriptor structs (`VersionDescriptor`, `ReleaseDesc`,
  …) carry `schema_version` (snake) on the wire; kebab-case structs (`Config`,
  `Secrets`, `VaultConfig`, `CoreComponent`) carry `schema-version` (kebab),
  achieved via `#[serde(rename = "schema-version")]` on the `VersionedX`-wrapper
  discriminator. The Rust field name stays `schema_version` (snake) for
  ergonomic Rust code in both cases.
- `versions/utils.rs` in cbscore-types carries only the `VersionType` enum at
  this stage; the parse-family bodies require `regex` and live in
  `cbscore::versions::utils` (Phase 2). Doctests on `VersionType` exercise serde
  round-trip; no parse-family doctests yet.

**Testable:** Per-type round-trip property tests
(`create → store → load == create`) on hand-crafted Rust values. `VersionType`
round-trips through serde with the documented lowercase wire representation.

## Commit 5 — schema_version wrappers + round-trip tests

Implement the `Versioned*` enum + `into_latest()` pattern for every wire format
named in design 002 § Wire-Format Versioning, plus the M0 round-trip acceptance
test corpus.

**Files:**

- `cbsd-rs/cbscore-types/src/config/versioned.rs` — `VersionedConfig` enum
  tagged on `schema-version` (kebab, per design 002 §Wire-Format Versioning;
  `Config` is a kebab-case struct), `into_latest()` method, public `Config`
  alias.
- Same pattern for `Secrets`, `VaultConfig`, `CoreComponent` (all kebab,
  `tag = "schema-version"`) and `ContainerDescriptor`, `VersionDescriptor`,
  `ImageDescriptor`, `ReleaseDesc`, `ReleaseComponent`, `BuildArtifactReport`
  (all snake, `tag = "schema_version"`). One module per format alongside its
  sibling type module.
- `cbsd-rs/cbscore-types/tests/fixtures/` — curated corpus of real-world YAML /
  JSON files copied from `cbscore/tests/` (Python side) and from operator
  deployments, **with the appropriate schema-version marker added as the first
  key (`schema-version: 1` for kebab-case YAML formats, `schema_version: 1` for
  snake-case descriptor formats) and the `type:` tag added to every git secret
  entry**. One subdirectory per wire format.
- `cbsd-rs/cbscore-types/tests/round_trip.rs` — integration test walking
  `tests/fixtures/`, parsing each file via the appropriate `Versioned*` wrapper,
  calling `into_latest()`, re-serialising, and asserting structural equality to
  the original.

**Design constraints:**

- Serde's internal-tag dispatch on integer-valued tags may need a hand-rolled
  `Deserialize` if string-matching does not accept integer-valued tags directly
  (design 002 lines 343–345). Decide at implementation time; the goal is
  unquoted integers on disk (e.g., `schema-version: 1` not
  `"schema-version": "1"`).
- An absent marker key is a hard error (`ConfigError::MissingSchemaVersion` or
  `VersionError::MissingSchemaVersion` depending on the format), not implicitly
  v0 (design 002 lines 284–285).
- Reading a marker integer higher than the compiled-in max is a hard error with
  an "upgrade cbscore-rs" message (design 002 lines 297–301).
- **Wire-key casing for the marker is pinned per format** (design 002
  §Wire-Format Versioning §Interaction with other wire-format decisions):
  - Kebab-case structs (`Config`, `Secrets`, `VaultConfig`, `CoreComponent`) use
    the wire key `schema-version`. Achieve this with
    `#[serde(rename = "schema-version")]` on the `VersionedX` enum discriminator
    and on the matching Rust field if one exists.
  - Snake-case descriptor structs (`VersionDescriptor`, `ReleaseDesc`,
    `ImageDescriptor`, `ReleaseComponent`, `ContainerDescriptor`,
    `BuildArtifactReport`) use the wire key `schema_version` as-is. Rust field
    names stay `schema_version` (snake) in both cases for ergonomic Rust code;
    the rename attribute handles the kebab YAML spelling.

**Testable:**

- `cargo test -p cbscore-types` passes.
- File-shape round-trip equality (`load → store → load == load`) holds for every
  file in `tests/fixtures/`. Contrasts with Commits 3 and 4's Rust-value
  round-trip (`create → store → load == create`): Commits 3/4 test value
  fidelity for hand-crafted Rust values; this commit tests file fidelity for
  real-world on-disk YAML/JSON corpus.
- Negative test: a `Config` YAML without `schema-version` (kebab) produces
  `ConfigError::MissingSchemaVersion`.
- Negative test: a `Config` YAML with `schema-version: 99` (kebab, above the
  compiled-in max) produces
  `ConfigError::UnknownSchemaVersion { found: 99, max_supported: 1 }`.
- Parallel negative tests on the descriptor side: a `VersionDescriptor` JSON
  without `schema_version` (snake) produces
  `VersionError::MissingSchemaVersion`; with `schema_version: 99` produces
  `VersionError::UnknownSchemaVersion`.
- Negative test: a `secrets.yaml` git entry without a `type:` tag produces a
  serde error citing the missing field.
